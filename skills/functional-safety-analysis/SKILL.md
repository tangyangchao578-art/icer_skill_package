# 功能安全分析技能

当用户需要进行功能安全分析和安全机制设计时，启用此技能。

## 使用时机

- 安全需求分解
- FMEDA 分析
- 安全机制设计
- 故障注入测试规划

## ISO 26262 基本概念

### ASIL 等级

- **QM**：质量管理，不要求安全机制
- **ASIL A**：低风险
- **ASIL B**：中低风险
- **ASIL C**：中高风险
- **ASIL D**：最高风险

### 安全生命周期

1. 概念阶段
2. 系统级开发
3. 硬件级开发
4. 软件级开发
5. 集成和验证
6. 功能安全评估
7. 生产发布

## FMEDA 分析流程

**F**ailure **M**odes, **E**ffects, and **D**iagnostic **A**nalysis

### 步骤

1. 列出所有模块
2. 识别每个模块的潜在失效模式
3. 分析每个失效模式对上层的影响
4. 分析对安全目标的影响
5. 分配安全机制
6. 计算诊断覆盖率
7. 计算 SPFM/LFM 度量

### 失效模式

-  stuck-at-0 / stuck-at-1
- 翻转错误
- 延迟错误
- 时序错误
- 连接错误

## 常用安全机制

### 单点故障检测

| 机制 | 适用场景 | 诊断覆盖率 |
|------|----------|------------|
| 奇偶校验 | 存储器总线 | 低 ~ 中 |
| ECC | 存储器 | 高 |
| 双锁双核 | 处理器内核 | 高 |
| 看门狗 | 程序运行 | 中 ~ 高 |
| 窗口看门狗 | 时钟监控 | 高 |
| 比较冗余 | 控制逻辑 | 高 |
| 时间冗余 | 计算重试 | 中 |

### 容错机制

- ECC 错误纠正
- 冗余模块切换
- 重试
- 进入安全状态

## 度量计算

### SPFM（Single-Point Fault Metric）

SPFM = (1 - Σ(λSP * (1 - DC)) / ΣλSP) * 100%

要求：
- ASIL B: ≥ 90%
- ASIL D: ≥ 99%

### LFM（Latent Fault Metric）

LFM = (1 - ΣλLP / Σλtotal) * 100%

要求：
- ASIL B: ≥ 60%
- ASIL D: ≥ 90%

## 设计准则

- 安全机制独立于功能逻辑
- 单点故障不能违反安全目标
- 共因故障必须被避免
- 安全机制不能被功能故障屏蔽
- 故障检测后必须进入安全状态

## 验证要求

- 每个安全机制必须被验证
- 故障注入测试验证检测能力
- 安全响应必须正确
- 诊断覆盖率必须测量
- 满足 ASIL 要求的度量

## 文档要求

- 安全需求规格
- 安全架构文档
- FMEDA 报告
- 安全验证报告
- 可追踪性矩阵（需求→实现→验证）
