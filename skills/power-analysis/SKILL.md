# 功耗分析和优化技能

当用户需要进行功耗分析和功耗优化时，启用此技能。涵盖从架构级到物理级的低功耗优化，包括 IR 降和电迁移分析。

## When to Activate

- 项目前期功耗预算评估
- 功耗分析结果解读和问题定位
- 低功耗设计优化
- IR 降问题分析和修复
- 电迁移问题分析和修复

## 功耗组成

### 动态功耗（主要部分，大多数设计占 80%+）

公式：
```
P_dynamic = α * C_load * Vdd² * f
```

- α = 开关活动性（单位时间翻转次数比例）
- C_load = 总负载电容
- Vdd = 电源电压
- f = 工作频率

**关键点：** 电压对功耗是平方关系，降低电压比降低频率更有效。

### 静态功耗

公式：
```
P_static = I_leakage * Vdd
```

- 漏电流导致
- 先进工艺（< 28nm）静态功耗占比显著增加
- 关断电源模块可以消除静态功耗

### 短路功耗

输入翻转时，PMOS 和 NMOS 同时导通一段时间导致的功耗。合理设计一般占 < 10%。

## 功耗分析流程

1. 获取开关活动性（仿真获取准确，或者默认值估计）
2. 提取每个网络负载电容（从布局布线结果）
3. 计算每个网络动态功耗
4. 计算每个单元漏电流静态功耗
5. 按模块汇总功耗
6. 生成功耗热图
7. 定位高功耗模块，开始优化

## 架构级功耗优化（效果最大）

| 优化方法 | 效果 | 说明 |
|----------|------|------|
| 降低频率 | 🟢 大 | 不影响性能就降低频率 |
| 降低电压 | 🟢 最大 | V² 关系，降低电压收益非常大 |
| 电源关断 | 🟢 大 | 不使用模块完全关断电源，消除静态动态功耗 |
| 多电压 | 🟡 中 | 不同模块不同电压，关键模块高电压，非关键低电压 |
| 时钟门控 | 🟡 中 | 关闭不使用模块时钟，节省动态功耗 |

**架构级优化收益 > RTL 级 > 物理级** → 越早优化收益越大。

## RTL 级功耗优化

### ✅ 门控时钟

每个寄存器都要输出使能，工具自动插入门控时钟：

```systemverilog
// ✅ 好：使能信号，工具可以插入门控
always_ff @(posedge clk or negedge rst_n) begin
  if (!rst_n) begin
    r_cnt <= '0;
  end
  else if (en) begin  // 只有使能为高才更新，工具可以门控时钟
    r_cnt <= r_cnt + 1'b1;
  end
end
```

❌ 不好：没有使能，一直翻转，不能门控。

### ✅ 操作数隔离

关闭不使用通路的逻辑翻转：

```
// 选通A通路的时候，B通路不翻转，节省功耗
if (sel == A) begin
  result = a_op;
end else begin
  result = b_op;
end
```

工具会自动隔离不使用通路，节省翻转功耗。

### ✅ 总线编码优化

- **地址总线/计数器**：格雷码编码 → 每次只有一位翻转 → 减少翻转活动性 → 节省功耗
- **数据总线**：总线翻转编码 → 减少翻转 → 节省功耗

**对比：二进制 vs 格雷码**

- 二进制计数从 0111 → 1000 → 四位都翻转 → 功耗大
- 格雷码计数从 0100 → 0101 → 只有一位翻转 → 功耗小

## 物理级功耗优化

- **移除死代码**：删除没有使用的逻辑，减少电容
- **缓冲优化**：不必要的大驱动换成小驱动，减少电容和短路功耗
- **线长优化**：关键路径线长优化，减少负载电容
- **电源网络优化**：降低 IR 降，不需要过高电压就能满足时序，降低整体功耗

## IR 降分析

### 什么是 IR 降

电源网络有电阻，电流流过产生电压降：

```
V_used = V_supply - I * R
```

### IR 降影响

- 局部电压降低 → 单元延迟增加 → 可能导致建立时间违例
- 严重 IR 降 → 电压太低 → 功能错误
- 大电流芯片 IR 降问题更严重

### IR 降检查标准

```
最大 IR 降 < 5% * Vdd
```
也就是说 1V 电源最大 IR 降 < 50mV。

### IR 降优化方法

1. **增加电源地线宽** → 降低电阻
2. **增加电源 strap 密度** → 降低电阻
3. **重新摆放模块** → 高功耗模块摆放在靠近电源 pad 的位置
4. **使用多层电源** → 上下层金属都是电源，降低电阻
5. **增加去耦电容** → 维持电压稳定

## 电迁移分析

### 什么是电迁移

大电流流过金属，电子撞击金属原子，长时间导致原子迁移，最终断路，芯片失效。

### 影响

- 影响芯片寿命
- 高电流线路更容易发生
- 电源地线路电流大，特别注意

### 电迁移检查标准

```
电流密度 < 最大允许电流密度（工艺给出）
```

### 电迁移优化方法

1. **增加线宽** → 降低电流密度
2. **并联多根金属线** → 分散电流，每根电流密度降低
3. **电源地使用上层金属** → 上层金属厚度大，电阻小，允许更大电流
4. **过孔并联** → 单个过孔电流密度不要超标

## 低功耗设计检查清单

- [ ] 总功耗满足项目预算要求
- [ ] 最大 IR 降 < 5% Vdd
- [ ] 所有网络电流密度满足电迁移要求
- [ ] 不使用模块都有电源关断
- [ ] 不使用模块时钟都有门控
- [ ] 计数器使用格雷码编码
- [ ] 高功耗模块已经优化

## 常见问题

### Q: 哪里最容易发生 IR 降问题？

A: 芯片中心距离电源 pad 最远，中心容易 IR 降大。高功耗模块放在中心容易问题。**推荐：** 高功耗模块均匀分布，靠近电源 pad。

### Q: 哪里最容易发生电迁移问题？

A: 电源 pad 下面的电源 straps，大电流输出 IO，电源环。这些地方电流最大。**推荐：** 这些地方线宽足够，过孔足够。

### Q: 电压降低 10% 能省多少功耗？

A: 功耗和电压平方成正比，电压 0.9Vdd → 功耗 (0.9)² = 0.81 → 省 19% 功耗 → 收益很大。

## 反模式（要避免）

❌ **不做功耗分析就流片** → 流片回来功耗超预算，芯片太热不能工作
❌ **忽略 IR 降** → 中心电压太低，时序不满足
❌ **电迁移不检查** → 产品出厂几个月就失效，可靠性问题
❌ **低功耗优化只在物理级做** → 架构级优化收益最大，早做更好

## 代理协作

- 使用 `physical-design-engineer` 代理进行物理设计实现
- 功耗优化在项目早期（架构阶段）开始做收益最大
- IR 降修复后重新做时序分析，因为延迟变化了
