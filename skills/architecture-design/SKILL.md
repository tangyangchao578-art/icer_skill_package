# 芯片架构设计技能

当用户需要进行芯片架构设计时，启用此技能。此技能提供芯片架构设计的完整指导，从需求定义到模块划分。

## 使用时机

- 新项目启动定义架构
- 新增功能模块定义接口
- 现有架构重构优化
- 评估不同架构方案

## 架构设计流程

### 1. 需求收集和分析

- **功能需求**：需要实现哪些功能
- **性能需求**：目标频率、吞吐量、延迟
- **面积预算**：门数、存储器容量
- **功耗预算**：动态功耗、静态功耗
- **接口需求**：片上接口、片外接口
- **安全需求**：功能安全 ASIL 等级、信息安全

### 2. 自上而下分解

1. **系统级**：整个芯片功能和接口
2. **子系统级**：划分为多个子系统
3. **模块级**：每个子系统划分为多个模块
4. **单元级**：每个模块划分为多个功能单元

### 3. 接口设计原则

- 使用标准总线协议（AXI, AHB, APB）
- 接口位宽匹配吞吐量需求
- 握手机制明确（ready/valid）
- 错误处理机制定义清楚
- 跨时钟域处理明确

### 4. 时钟和复位规划

- **时钟域划分**：不同频率不同时钟域
- **频率选择**：满足性能，不要过度超频
- **复位策略**：异步复位还是同步复位
- **复位域划分**：哪些模块需要一起复位

### 5. 存储器规划

- 使用存储器大小和数量估算
- 存储器位置摆放规划
- 访问端口数量规划
- ECC/奇偶校验需求

### 6. 电源规划

- 电源域划分
- 动态电源关断需求
- 电平转换器位置
- 隔离单元位置

### 7. DFT 规划

- 扫描链规划
- 存储器 BIST 需求
- JTAG 接口位置
- 测试点插入需求

## 架构评估指标

| 指标 | 评估方法 |
|------|----------|
| 可验证性 | 是否容易搭建验证环境 |
| 可实现性 | 是否满足面积时序预算 |
| 可调试性 | 是否容易调试内部状态 |
| 可扩展性 | 是否方便未来扩展 |
| 功耗 | 是否满足功耗预算 |
| 面积 | 是否满足面积预算 |
| 时序 | 是否满足时序预算 |
