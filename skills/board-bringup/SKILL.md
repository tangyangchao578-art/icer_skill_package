# FPGA 原型和板级 Bringup 技能

当用户需要进行 FPGA 原型验证或硅后板级验证时，启用此技能。

## 使用时机

- FPGA 原型项目启动
- 新 PCB 第一次上电
- 硅后芯片第一次测试
- 板级问题调试

## Bringup 顺序（从简单到复杂）

### 阶段 1：静态检查（不开电）

- [ ] 检查电源和地是否短路（万用表测量）
- [ ] 检查所有电源轨导通性
- [ ] 检查晶体/振荡器连接
- [ ] 检查 JTAG/SWD 连接
- [ ] 检查所有器件焊接正确

### 阶段 2：第一次上电

- [ ] 设置电源电流限制（比如 500mA）
- [ ] 缓慢上升电压
- [ ] 测量静态电流
- [ ] 检查所有电源轨电压正确
- [ ] 检查器件温度不异常

### 阶段 3：时钟和复位测试

- [ ] 测量参考时钟频率（示波器）
- [ ] 检查 PLL 锁定
- [ ] 输出时钟测量
- [ ] 复位信号电平正确
- [ ] 复位释放时序正确

### 阶段 4：配置接口测试

- [ ] JTAG/SWD 能够连接
- [ ] 能够读取 ID 码
- [ ] 能够加载配置比特流（FPGA）
- [ ] 能够擦除和编程 flash
- [ ] 基本寄存器读写正确

### 阶段 5：模块逐个测试

- 从最简单模块开始：
  1. GPIO 测试
  2. UART 测试
  3. I2C/SPI 测试
  4. 存储器测试
  5. 复杂模块测试

### 阶段 6：系统集成测试

- 模块间接口测试
- 完整应用场景测试
- 压力测试
- 温度频率极限测试

## 常用调试手段

### 片上观测（FPGA）

- ILA (Integrated Logic Analyzer)：抓取内部信号
- VIO (Virtual Input/Output)：动态改变输入
- 直接修改寄存器查看输出
- 触发条件设置好，一次抓取成功

### 外部仪器

- **示波器**：测量时钟，电源，重要信号
- **逻辑分析仪**：捕获并行总线，协议分析
- **万用表**：测量电压电流电阻
- **红外相机**：测量芯片温度分布

### 软件调试

- UART 打印调试信息
- 寄存器读写测试
- 内存测试
- 简单测试程序反复运行

## 常见问题定位

### 问题：电源电流过大

- 可能原因：短路，器件焊错，漏电
- 排查方法：断电测量电阻，逐个上电，手摸温度

### 问题：JTAG 连接不上

- 可能原因：TMS/TCK/TDI/TDO 接反，复位不对，电源不对
- 排查方法：检查连线，测量时钟，检查复位

### 问题：PLL 不锁定

- 可能原因：参考频率不对，范围超出，配置不对，电源不对
-排查方法：检查参考时钟频率，检查配置寄存器，检查电压

### 问题：功能不对，但寄存器读写正确

- 可能原因：配置不对，复位没释放，时钟不对
- 排查方法：读回配置确认，检查时钟使能，检查复位状态

### 问题：偶尔出错

- 可能原因：时序余量不够，亚稳定，电源噪声
- 排查方法：降低频率看是否还错，检查电源纹波，增加同步级

## FPGA 特定注意事项

- 检查资源使用（LUT, FF, BRAM, DSP）不超过器件容量
- 检查时序收敛（建立保持满足）
- 引脚分配正确，不冲突
- 使用正确的 IO 标准和电压
- 预留调试引脚方便调试

## 硅后特定注意事项

- 第一片芯片电流异常不要继续加电
- 逐步测试，从简单到复杂
- 记录每一步结果，方便回溯
- 失效芯片做好标记保存，方便失效分析

## 文档要求

- 记录测试环境
- 记录测试步骤
- 记录每个测试结果
- 记录发现的问题
- 记录根因分析
- 记录修复方法和验证结果
