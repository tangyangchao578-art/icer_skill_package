# FPGA 原型和板级 Bringup 技能

当用户需要进行 FPGA 原型验证或硅后板级验证时，启用此技能。提供从静态检查到系统测试的完整 bringup 流程和问题调试方法。

## When to Activate

- 新 FPGA 原型项目启动
- 新 PCB 第一次上电 bringup
- 硅后芯片第一次测试
- 板级功能问题调试
- 量产前验证测试

## Bringup 顺序原则：**从简单到复杂**

永远从最简单的开始，一步一步往前走，一个测试通过再下一个。

---

### 阶段 1：静态检查（不开电）

**一定要做，不开电检查出问题比上电烧器件好。**

- [ ] 目视检查 PCB：有没有明显短路，有没有器件焊反，有没有少件
- [ ] 万用表测量电源和地之间电阻 → 确认没有短路
- [ ] 检查所有电源轨导通性 → 每个电源都能通到芯片
- [ ] 检查晶体/振荡器焊接正确
- [ ] 检查 JTAG/SWD 信号线连接正确
- [ ] 检查所有无源器件值正确焊接
- [ ] 检查接插件焊接正确

---

### 阶段 2：第一次上电

**安全第一，电流限制，慢慢升压。**

- [ ] 电源设置电流限制（通常 100mA - 500mA 起始）
- [ ] 关断所有开关，不加载配置
- [ ] 缓慢上升电压到目标电压
- [ ] 测量静态电流 → 和预期比较
- [ ] 测量每个电源轨电压 → 确认电压正确，压降不大
- [ ] 手摸各个器件 → 有没有异常发热
- [ ] 电流过大立即断电，回去检查短路

---

### 阶段 3：时钟和复位测试

- [ ] 示波器测量参考时钟频率 → 频率正确，幅度正确
- [  配置 PLL，测量 PLL 锁定 → 锁定指示灯亮，频率正确
- [ ] 测量各个输出时钟频率 → 都正确
- [ ] 测量复位信号电平 → 复位时正确电平，释放后正确电平
- [ ] 检查复位释放时序 → 正确顺序

---

### 阶段 4：配置接口测试

- [ ] JTAG/SWD 能够连接 → 能够识别芯片
- [ ] 能够读取 ID 码 → ID 正确
- [ ] FPGA：能够加载配置比特流 → 成功
- [ ] 能够擦除和编程 flash → 成功
- [ ] 基本寄存器读写测试 → 读写正确

**如果这一步过不去，不要往下走，一定先解决。**

---

### 阶段 5：模块逐个测试

**从最简单到最复杂，一个模块测试通过再下一个。**

1. **GPIO 测试** → 输入输出读写正确
2. **UART 测试** → 能发能收
3. **I2C/SPI 测试** → 能够读写外设
4. **存储器测试** → 读写每个地址，没有错误
5. **DDR 测试** → 读写测试，带宽测试
6. **复杂模块测试** → 核心功能测试

---

### 阶段 6：系统集成测试

- [ ] 模块间接口测试 → 模块互连正确
- [ ] 完整应用场景测试 → 功能正确
- [ ] 压力测试 → 长时间运行，温度升高有没有问题
- [ ] 极限测试 → 最高频率，最低电压，高低温

## 常用调试手段

### 片上观测（FPGA 推荐）

- **ILA (Integrated Logic Analyzer)** → 抓取内部信号波形
  - 设置好触发条件，一次抓取成功
  - 触发点放在错误点
  - 足够深度存储波形
- **VIO (Virtual Input/Output)** → 动态改变输入信号，观察输出
- 直接从寄存器读写 → 改变配置不用重新综合
- 所有重要状态都能引出观测 → 方便调试

### 外部仪器

- **示波器** → 测量时钟频率，电压纹波，信号边沿
- **逻辑分析仪** → 捕获并行总线，协议解码
- **万用表** → 直流电压，电流，电阻
- **红外热像仪** → 测量芯片温度分布，找到过热区域

### 软件调试

- UART 打印调试信息 → 简单有效
- 内存测试 → 读写每个位置检查错误
- 循环测试 → 复现偶发错误
- 寄存器回读 → 确认配置真的正确写入

## 常见问题定位

### 问题 1：电源电流过大

**可能原因：**
- 电源地短路 → 最常见
- 器件焊错 → 比如稳压管反了
- 漏电 → 芯片坏了
- 负载过重 → 某个模块一直拉电流

**排查方法：**
1. 断电，万用表测量电源地电阻 → 电阻太小就是短路
2. 逐个移除负载，断开电源域 → 找到哪个区域短路
3. 上电后手摸 → 发烫的器件就是问题点

### 问题 2：JTAG/SWD 连接不上

**可能原因：**
- TCK/TMS/TDI/TDO 接线错
- 电源不对，芯片没工作
- 复位不对，芯片一直复位
- 速度太快，适配器不支持

**排查方法：**
1. 测量每个信号线连接 → 从接口到芯片引脚通
2. 测量电源电压正确
3. 测量复位电平正确
4. 降低 JTAG 速度试试

### 问题 3：PLL 不锁定

**可能原因：**
- 参考时钟频率不对 → 超出 PLL 范围
- 参考电容不对 → 不起振
- 电压不对 → 不能锁定
- 配置不对 → 分频系数错了

**排查方法：**
1. 示波器测量参考时钟 → 频率正确
2. 测量电源电压正确
3. 检查配置寄存器读回 → 配置正确

### 问题 4：功能不对，但寄存器读写正确

**可能原因：**
- 配置不对 → 软件配置错了
- 复位没释放 → 芯片一直复位
- 时钟没开 → 模块时钟关了
- 时钟频率不对 → 太快太慢

**排查方法：**
1. 回读配置寄存器 → 确认配置正确
2. 测量模块时钟 → 频率正确，时钟开门
3. 检查复位状态 → 已经释放
4. 降低频率试试 → 如果好了就是时序不够

### 问题 5：偶尔出错，不是每次都错

**可能原因：**
- 时序余量不够 → 偶尔建立时间错
- 亚稳态 → 跨时钟域同步不好
- 电源噪声 → 电压波动
- 温度影响 → 温度高了出错

**排查方法：**
1. 降低工作频率 → 如果降频率就好了，确定是时序问题
2. 增加跨时钟域同步级数 → 从两级改三级试试
3. 测量电源纹波 → 纹波太大稳压不好
4. 降温试试 → 如果降温好就是温度问题

## FPGA 特定注意事项

- [ ] 检查资源使用：LUT, FF, BRAM, DSP 不超过器件容量 → 预留 20% 余量
- [ ] 检查时序收敛：建立保持都满足
- [ ] 引脚分配正确：没有冲突，正确连接到外设
- [ ] 使用正确 IO 标准和电压 → 电压不对会不工作
- [ ] 预留调试引脚 → 方便ILA 观测内部信号
- [ ] 检查功耗 → 满足散热要求

## 硅后芯片特定注意事项

- [ ] 第一片芯片电流异常 → 立即断电，不要继续加电
- [ ] 逐步测试，从简单到复杂 → 不要一开始就跑复杂测试
- [ ] 记录每一步测试结果 → 方便回溯，方便失效分析
- [ ] 失效芯片做好标记保存 → 不要扔，后面失效分析要用
- [  第一片不要跑极限压力 → 逐步增加

## 调试方法论

### 二分法定位

- 问题在前半部分还是后半部分 → 砍掉一半，缩小范围
- 重复，直到找到问题模块

### 隔离法

- 禁用所有不相关模块 → 只保留问题相关模块
- 排除其他模块影响

### 对比法

- 和仿真波形对比 → 看哪个信号不对
- 和已知好的板子对比 → 看差异

## 文档要求

bringup 过程记录：

- [ ] 记录测试环境（工具版本，仪器，温度）
- [ ] 记录每个测试步骤
- [ ] 记录每个测试结果（pass/fail）
- [ ] 记录发现的问题
- [ ] 记录根因分析
- [ ] 记录修复方法和验证结果

方便以后类似问题快速定位，也方便下个版本改进。

## 检查清单

bringup 完成检查：

- [ ] 所有电源电压正确
- [ ] 所有时钟频率正确
- [ ] 配置接口工作正常
- [ ] 所有模块测试通过
- [ ] 系统集成测试通过
- [ ] 压力测试通过
- [ ] 极限测试通过
- [ ] 所有问题都已经修复验证

## 反模式（要避免）

❌ **上来就跑完整系统** → 出问题不知道哪错了，从头来吧
❌ **不记录结果** → 一周后自己都忘了做到哪了
❌ **电流过大继续加电** → 直接烧坏器件，没救
❌ **跳过分步测试** → 一下子改很多，找不到问题
