# IC 前端设计 - IC 特定扩展

## RTL 编码 IC 特定准则

### 可综合性

IC 设计对可综合性要求更严格：

- [ ] 不使用 for-generate 之外的生成语句不当
- [ ] 不使用动态位选择在组合逻辑中
- [ ] 不使用不确定位宽
- [ ] 避免使用过于复杂的 case 语句（综合会产生太大的多路选择器）

### 时序收敛友好

- [ ] 避免太长的组合逻辑路径
- [ ] 流水线分割长路径
- [ ] 输出从寄存器出，不直接从组合逻辑出
- [ ] 大扇出信号寄存器分割
- [ ] 避免回环（综合工具很难处理）

### 面积优化

- [ ] 不使用不必要的寄存器
- [ ] 复用资源，减少面积
- [ ] 使用现成的工艺库乘法器/加法器
- [ ] 状态编码：One-hot 适合高速，格雷码适合低功耗，二进制适合小面积

### 功耗优化

- [ ] 避免不必要的翻转
- [ ] 门控时钟关闭不使用的模块
- [ ] 操作数隔离
- [ ] 使用低功耗编码（格雷码比二进制低功耗）

## 跨时钟域设计准则

### 单控制信号（慢到快）

- 两级同步：必须打两拍，不能只打一拍
- 正确实例：两级寄存器同步
- 错误实例：一级寄存器，亚稳态无法解决

### 单控制信号（快到慢）

- 脉冲展宽 + 两级同步
- 或者使用握手机制
- 不能直接传输脉冲信号

### 总线信号

- 使用握手机制：请求-应答
- 或者使用异步 FIFO
- 不能直接传送多位信号
- 格雷码编码地址降低亚稳态输出概率

### CDC 检查

- 所有跨时钟域必须标记给工具检查
- 运行 CDC 检查确认没有问题
- 人工检查同步电路正确性

## 复位设计 IC 特定

### 异步复位

优点：复位释放不依赖时钟，适合没有时钟的模块
缺点：容易受到复位毛刺影响，需要同步释放

### 同步复位

优点：不会有毛刺，适合FPGA
缺点：需要时钟才能复位，面积更大

### 项目选择

在同一个项目中保持一致，不要混合使用两种风格。
