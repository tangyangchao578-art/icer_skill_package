# IC 前端验证 - IC 特定扩展

## IC 验证特定准则

### 验证计划

IC 项目必须有正式的验证计划：

- [ ] 列出所有功能点
- [ ] 定义每个功能点的验证方法
- [ ] 定义覆盖率目标
- [ ] 定义验收标准
- [ ] 进度跟踪

### 定向测试 vs 随机测试

- **定向测试**：验证特定功能点，边界情况
- **随机测试**：验证大空间，寻找 corner case
- **约束随机验证**：在约束空间内随机激励，提高覆盖率

### 覆盖率驱动验证

- 根据覆盖率结果添加遗漏的测试用例
- 功能点没有覆盖，添加对应的测试
- 代码覆盖率没有覆盖，分析原因
- 达到覆盖率目标才能收敛

## UVM IC 设计准则

- [ ] 使用正确的 phase 机制
- [ ] 使用 sequence 分层产生激励
- [ ] 使用 config_db 传递配置
- [ ] 使用 objection 机制控制仿真结束
- [ ] 使用 register 模型自动化访问寄存器
- [ ] 使用 RAL（Register Abstraction Layer）

## 形式验证应用

- 等价性检查：RTL vs 网表
- 属性检查：检查关键接口属性
- 芯片整体太复杂，一般用于模块级验证
- 发现仿真难以发现的 corner case

## 功耗验证

- 检查门控时钟是否正确工作
- 检查电源关断是否正确
- 检查隔离单元是否正确
- 检查状态保持是否正确

## 形式验证应用场景

1. 等价性检查：综合前后
2. 等价性检查：ECO 前后
3. 属性检查：总线接口协议
4. 属性检查：状态机安全属性

## CDC 验证

- 检查所有跨时钟域路径
- 检查同步电路是否正确
- 检查亚稳定性平均故障间隔时间
- 检查脉冲展宽是否正确
