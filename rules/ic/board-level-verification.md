# 板级验证 - IC 特定扩展

## FPGA 原型验证特定

### FPGA 选型考虑

- 足够的逻辑资源（LUT, FF）
- 足够的存储器资源（BRAM）
- 足够的 DSP 资源
- 足够的 GPIO 数量
- 满足速度等级要求
- 预留 20% 资源余量用于后续调试

### 移植注意事项

- ASIC 中的异步复位在 FPGA 同样工作
- ASIC 中的门控时钟在 FPGA 需要重新考虑
- 使用 FPGA PLL/MMCM 产生高质量时钟
- 使用正确的 IO 标准
- 考虑 FPGA 电源散热

### 加速仿真

- 把仿真中太慢的部分放到 FPGA 原型运行
- JTAG 调试接口方便访问内部寄存器
- 高速接口接出去可以和真实系统对接
- 提供软件运行环境，软硬件协同仿真

## 硅后验证特定

### 第一批芯片测试重点

- 电源地是否短路
- 时钟是否正确输出
- JTAG/SWD 是否能访问
- 寄存器读写是否正确
- 基本功能是否正常
- 功耗是否在预算内

### 测试顺序

从简单到复杂：

1. 静态测试（不开时钟）
2. 时钟测试
3. 复位测试
4. 寄存器接口测试
5. 模块功能测试
6. 集成测试
7. 系统测试
8. 极限测试（频率、电压、温度）

### 良率分析

- 记录每个芯片的测试结果
- 分类失败原因
- 统计良率
- 分析系统性问题
- 提供改机建议

### 失效分析

- 失效芯片进行失效分析
- 找到物理失效位置
- 分析根因
- 提供设计改进建议

## 调试技巧 IC 特定

- 首次上电电流过大：立即断电，检查短路
- 时钟不出：检查 PLL 配置，参考时钟输入
- JTAG 不能访问：检查 TCK/TMS/TDI/TDO 连接，复位
- 功能不对：先读寄存器确认配置是否正确，再看数据流
- 使用片上观测模块（ILA）查看内部信号
