# 后端设计原则

## 目的

定义布局布线、时序收敛、物理验证阶段的设计原则，确保设计满足所有物理约束并能成功流片。

## 后端设计流程

1. **Floorplan**：布图规划，电源规划，模块放置
2. **Placement**：标准单元布局
3. **CTS**：时钟树综合
4. **Routing**：布线
5. **Timing Closure**：时序收敛
6. **Physical Verification**：物理验证

## Floorplan

### 基本原则

- [ ] 根据数据流安排模块位置，减少长连线
- [ ] 电源地网络满足 IR 降和电迁移要求
- [ ] 留出足够的 IO 摆放空间
- [ ] 去耦电容布置合理
- [ ] 考虑供电引脚位置，减少电源路径电阻

### PDN（电源分布网络）

- [ ] 满足电压降要求：IR 降 < 5% 核心电压
- [ ] 满足电迁移要求：电流密度在工艺允许范围内
- [ ] 多层电源地叠层降低阻抗
- [ ] 使用电源带隔离不同模块电源域

## Placement

### 布局优化

- [ ] 减少关键路径连线长度
- [ ] 控制布线拥塞，避免局部拥塞过高
- [ ] 数据通路模块放置紧凑
- [ ] 控制线长，减少电容负载
- [ ] 优化摆放减少跨区块长连线

## CTS（时钟树综合）

### 时钟树目标

- [ ] 满足时钟偏差要求（skew）
- [ ] 满足时钟插入延迟要求（latency）
- [ ] 最小化时钟树功耗
- [ ] 满足最大转换时间要求
- [ ] 减少时钟网络毛刺

### 时钟设计准则

- [ ] 时钟不应该经过逻辑门（除了时钟多路选择器）
- [ ] 不同时钟域隔离
- [ ] 门控时钟的使能信号同步
- [ ] 避免时钟线上有太多负载

## Routing

### 布线准则

- [ ] 关键路径优先布线
- [ ] 时钟网络优先布线
- [ ] 避免天线效应，插入天线二极管
- [ ] 满足最小间距要求
- [ ] 满足线宽要求（大电流）
- [ ] 差分对布线等长等间距

## Timing Closure

### 时序优化

- [ ] 建立时间（setup）满足要求
- [ ] 保持时间（hold）满足要求
- [ ] 修正最大转换时间违反
- [ ] 修正最大电容负载违反
- [ ] 使用有用偏斜优化时序
- [ ] 进行 ECO 修改解决最后时序违反

### 检查清单

- [ ] 所有路径都满足建立时间要求
- [ ] 所有路径都满足保持时间要求
- [ ] 时序报告没有未检查的例外路径
- [ ] OCv 时序分析满足要求
- [ ] PVT 变化下时序仍然闭合

## Physical Verification

### 必须进行的检查

- [ ] **DRC**：设计规则检查，零违反
- [ ] **LVS**：版图和原理图一致，零违反
- [ ] **ERC**：电学规则检查，零违反
- [ ] **天线效应检查**：零违反
- [ ] **密度检查**：满足金属密度要求
- [ ] **填充金属**：满足密度要求

### ECO 修改后必须重复检查

- [ ] LVS 重新检查
- [ ] DRC 重新检查
- [ ] 连接性检查

## 功耗优化（后端）

- [ ] 多电压域设计正确隔离
- [ ] 电源开关正确插入
- [ ] 保留状态寄存器正确连接
- [ ] 电平转换器正确使用

## 工具使用

- **布局布线**：IC Compiler II, Innovus, OpenROAD
- **时序分析**：PrimeTime, Tempus
- **物理验证**：ICV, PVS, Calibre
- **IR 降分析**：RedHawk, Voltus

## 代理使用

- 后端实现：使用 **physical-design-engineer** 代理
- DRC/LVS 调试：使用 **drc-lvi-debug** 技能
- 时序分析：使用 **timing-analysis** 技能
