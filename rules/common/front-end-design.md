# 前端设计原则

## 目的

定义 RTL 前端设计的最佳实践，确保设计可综合、可验证、可维护。

## 基本准则

### 可综合性

- [ ] 使用可综合的 RTL 编码风格
- [ ] 避免使用初始块（initial block）给寄存器赋初值（除了仿真）
- [ ] 避免使用行为级延迟（#delay）
- [ ] 使用标准的时序逻辑模板

### 编码风格

- [ ] 有意义的命名：使用小写字母加下划线（snake_case）
- [ ] 常量使用大写字母加下划线
- [ ] 每个模块只包含一个顶层模块
- [ ] 避免深层嵌套（不超过 4 层）
- [ ] 一行只声明一个信号

### 时序设计

- [ ] 所有输出来自寄存器（流水线）
- [ ] 明确时钟域和复位域
- [ ] 正确处理异步复位和同步复位
- [ ] 跨时钟域必须有明确的同步电路
- [ ] 避免使用门控时钟（工具自动处理更好）

### 复位设计

- [ ] 一致的复位策略：整个设计选择异步复位或同步复位
- [ ] 异步复位：释放必须同步到时钟
- [ ] 数据通路寄存器：必须可复位
- [ ] 流水线寄存器：必须可复位
- [ ] 不修改状态的存储器：可以不复位

## 设计检查清单

在提交代码前检查：

- [ ] 所有输入输出端口都有正确的位宽
- [ ] 所有信号都已定义，没有隐式端口
- [ ] 组合逻辑没有锁存（不完全条件判断）
- [ ] 状态机有合法状态处理（default case）
- [ ] 乘法、除法等运算器满足时序要求
- [ ] 大扇出信号已经分段或寄存器输出
- [ ] 符合项目编码风格规范

## 工具使用

- **lint 检查**：运行 SpyGlass 或 Verilator 进行代码质量检查
- **CDC 检查**：跨时钟域必须进行 CDC 分析
- **综合**：运行逻辑综合检查是否可综合

## 代理使用

- RTL 实现：使用 **rtl-designer** 代理
- 代码质量检查：使用 **code-reviewer** 代理
