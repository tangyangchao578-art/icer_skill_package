# 板级验证原则

## 目的

定义 FPGA 原型验证和硅后板级验证的基本原则，确保验证高效、问题定位准确。

## 原型验证流程

### 1. 前期准备

- [ ] 验证 PCB 设计正确性（连线、电源、接地）
- [ ] 准备配置文件（比特流、固件）
- [ ] 准备测试程序和测试向量
- [ ] 检查测量仪器（示波器、逻辑分析仪）

### 2. 上电顺序

严格遵循上电顺序：

1. 连接电源和地
2. 设置电压电流限制
3. 缓慢升电压
4. 测量静态电流
5. 加载配置
6. 开始测试

### 3. 测试层次

1. **基本测试**：时钟、复位、JTAG/SWD 接口
2. **模块测试**：逐个测试每个模块功能
3. **集成测试**：测试模块间交互
4. **系统测试**：运行完整应用场景
5. **压力测试**：极限频率、极限温度测试

## 调试方法

### 问题定位

- **分层排查**：从硬件到软件，从底层到顶层
- **二分法**：逐步缩小问题范围
- **隔离法**：禁用不相关模块，只保留问题相关部分
- **对比法**：与仿真波形对比

### 常用调试手段

- [ ] 使用 ILA/VIO 进行片上观测
- [ ] 使用逻辑分析仪捕获接口信号
- [ ] 使用示波器测量时钟和电源
- [ ] 使用 UART/print 输出调试信息
- [ ] 使用 JTAG 访问内部寄存器

## FPGA 特定注意事项

- [ ] 检查 FPGA 引脚分配是否正确
- [ ] 检查约束文件（时序约束、引脚约束）
- [ ] 评估资源使用（LUT、FF、BRAM、DSP）
- [ ] 检查时序收敛
- [ ] 使用适当的时钟管理（PLL/MMCM）

## 硅后验证特定注意事项

- [ ] 首先检查电源和地是否短路
- [ ] 检查ESD保护二极管是否正常导通
- [ ] 逐步测试不同电压域
- [ ] 检查bonding是否正确
- [ ] 温度测试：低温和高温验证

## 文档要求

- [ ] 记录测试环境配置
- [ ] 记录测试用例和测试结果
- [ ] 记录发现的问题和根因分析
- [ ] 记录修复方法和验证结果

## 代理使用

- 板级调试：使用 **validation-engineer** 代理
- 测试程序开发：使用 **eda-scripting** 技能
