# 中端设计原则

## 目的

定义逻辑综合、DFT 和物理综合阶段的设计原则，确保设计满足面积、时序、功耗要求。

## 逻辑综合

### 综合流程

1. **读入设计**：读入 RTL 代码和 IP
2. **链接设计**：解析层次和模块连接
3. **约束加载**：加载时序约束和物理约束
4. **逻辑优化**：高级优化和门级优化
5. **映射**：映射到目标工艺库
6. **输出网表**：输出用于后端设计的网表

### 约束设计原则

- [ ] 提供准确的时钟约束（周期、不确定性、抖动）
- [ ] 提供准确的输入输出延迟
- [ ] 设置正确的时钟域
- [ ] 设置 false path 和 multicycle path
- [ ] 不要过度约束，也不要欠约束
- [ ] 时钟不确定性设置合理

### 优化策略

- **面积优化**：优先减小面积
- **速度优化**：优先满足最高频率
- **功耗优化**：优先降低动态功耗
- **根据设计目标选择合适的优化策略**

## DFT（可测试性设计）

### DFT 需求

- [ ] 扫描链：所有同步单元都能接入扫描链
- [ ] 边界扫描：JTAG/IEEE 1149.1
- [ ] 存储器内建自测试（MBIST）
- [ ] 逻辑内建自测试（LBIST）
- [ ] 测试点插入：提高难测故障的可观测性

### DFT 检查清单

- [ ] 扫描覆盖率满足要求（> 95%）
- [ ] 所有异步复位都能被扫描控制
- [ ] 三态输出在测试模式下可控
- [ ] 测试模式不会导致功耗过载
- [ ] 扫描链长度平衡

## 物理综合

### 物理综合流程

- [ ] 基于布局的逻辑优化
- [ ] 缓冲插入优化
- [ ] 网表重构优化
- [ ] 时序优化考虑实际布线拥塞

### 物理综合检查清单

- [ ] 保持原始的 RTL 层次结构（便于调试）
- [ ] 不改变功能（等价性检查通过）
- [ ] 满足时序要求
- [ ] 面积增量在预算范围内

## 等价性检查

- [ ] 综合后必须进行形式验证
- [ ] 物理综合后必须进行形式验证
- [ ] ECO 后必须进行形式验证
- [ ] 确保修改不会改变功能

## 工具使用

- **逻辑综合**：Design Compiler, Genus, Yosys
- **DFT**：DFT Compiler, TetraMAX
- **物理综合**：Design Compiler Graphical, Genus Physical Synthesis
- **等价性检查**：Formality, Conformal

## 代理使用

- 综合约束编写：使用 **eda-scripting** 技能
- 问题排查：使用 **build-error-resolver** 代理
