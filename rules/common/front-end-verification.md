# 前端验证原则

## 目的

定义功能验证的方法论，确保设计质量，找出尽可能多的设计缺陷。

## 验证方法论

### 验证层次

1. **模块级验证**：验证单个模块功能
2. **子系统级验证**：验证多个模块集成
3. **系统级验证**：验证整个芯片功能
4. **回归验证**：每次代码修改后自动运行

### 验证平台组件

标准验证平台应包含：

- **测试用例**：不同功能场景
- **驱动器**：驱动 DUT 输入
- **监视器**：观测 DUT 输出
- **检查器**：自预测结果正确性
- **覆盖率收集**：功能覆盖和代码覆盖
- **断言**：关键时序和接口属性

## UVM 验证准则

- [ ] 遵循 UVM 编码规范
- [ ] 使用 factory 机制创建对象
- [ ] 使用 phase 机制控制仿真流程
- [ ] 配置数据库传递配置信息
- [ ] 使用 sequence 生成激励
- [ ] 订阅 analysis port 获取观测数据

## 断言验证

- [ ] 在接口放置协议断言
- [ ] 在状态机放置状态转换断言
- [ ] 使用 SVA（SystemVerilog Assertion）描述属性
- [ ] 复位释放后，所有断言应能正确重置
- [ ] 区分立即断言和并发断言

## 覆盖率

### 必须达到的覆盖率目标

- **代码覆盖率**：
  - 行覆盖率：>= 90%
  - 翻转覆盖率：>= 85%
  - 条件覆盖率：>= 80%
  - FSM 覆盖率：100%

- **功能覆盖率**：
  - 所有功能点都被覆盖
  - 所有交叉场景都被覆盖
  - 边界情况被覆盖

## 检查清单

验证完成前检查：

- [ ] 所有测试用例都通过
- [ ] 覆盖率目标已达到
- [ ] 所有断言都没有失败
- [ ] 没有内存泄漏
- [ ] 所有错误注入测试都正确处理
- [ ] 异常情况处理验证完成

## 工具支持

- **仿真器**：VCS, Xcelium, Questa, Verilator
- **覆盖率分析**：Verdi
- **形式验证**：Synopsys Formality, Cadence Conformal

## 代理使用

- 验证平台开发：使用 **verification-engineer** 代理
- 断言设计：使用 **assertion-based-verification** 技能
